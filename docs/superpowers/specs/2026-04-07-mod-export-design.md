# SkinTatoo Mod 导出功能设计

**日期**: 2026-04-07
**状态**: 设计已批准，待生成实施计划

## 背景与目标

SkinTatoo 当前通过 Penumbra **临时 mod**（`AddTemporaryModAll` IPC）实现实时预览：合成纹理写到 `outputDir/preview_*.tex`，emissive 写到 `preview_*.mtrl`，再通过 IPC 注册 `gamePath → diskPath` 字典，并由 `EmissiveCBufferHook` 实时更新 CBuffer 中的 `g_EmissiveColor`。

但临时 mod 仅在游戏会话内有效，无法分享、无法持久。本功能新增"导出 Mod"能力，把当前工程冻结成一个可独立分发的 Penumbra mod。

**两个独立入口：**
1. **导出到本地** — 生成单文件 `.pmp` 保存到用户指定路径
2. **安装到 Penumbra** — 生成 `.pmp` 后调 `InstallMod` IPC 让 Penumbra 直接安装

## 用户决策摘要

| # | 决策点 | 选择 |
|---|--------|------|
| 1 | 导出范围 | 用户在导出对话框里勾选 group |
| 2 | Emissive 处理 | 把当前颜色×强度烘焙进 .mtrl 的 `g_EmissiveColor` |
| 3 | 导出 UI | 两个独立按钮：本地 / Penumbra |
| 4 | 本地格式 | `.pmp` 单文件（zip） |
| 5 | Penumbra 安装途径 | `InstallMod(pmpPath)` IPC（与本地导出共用 .pmp 打包） |
| 6 | meta.json 元信息 | 默认值存 Configuration，导出对话框可临时覆盖 |
| 7 | 合成内容 | 仅 visible 图层参与合成 |

### 选定方案：复用 PreviewService 合成路径

`ModExportService` 调用 `PreviewService.CompositeForExport(group, stagingDir)`，后者复用现有的 `LoadBaseTexture` / `CpuUvComposite` / `CompositeEmissiveNorm` / `WriteBgraTexFile` / `MtrlFileWriter.WriteEmissiveMtrl`，但把目标路径改成 staging 目录、只对 visible 图层合成，且**不**触碰 GPU swap、cache、hook 等运行时状态。

**已拒绝的备选**：
- *完全独立的导出管线*：合成代码两份，参数语义/blend 模式只要漏改一边就让"预览"和"导出"不一致 — 风险过高
- *把合成抽成纯静态库*：重构面积大，涉及 baseTextureCache / emissiveOffsets 等当前是 PreviewService 字段的状态 — 超出本任务范围

## 组件职责

### `ModExportOptions` (Core)

```csharp
public enum ExportTarget { LocalPmp, InstallToPenumbra }

public class ModExportOptions
{
    public string ModName = "";
    public string Author = "";
    public string Version = "1.0";
    public string Description = "";
    public List<TargetGroup> SelectedGroups = new();
    public ExportTarget Target;
    public string? OutputPmpPath;  // 仅 LocalPmp 时填
}
```

### `ModExportWindow` (Gui)

ImGui 弹窗，以 `(标题, ExportTarget)` 元组为状态切换：
- 标题"导出到本地" → 含"浏览…"按钮调起文件保存对话框
- 标题"安装到 Penumbra" → 不显示文件路径输入

**字段**：
- mod 名称（默认 = `DecalProject` 名或"SkinTatoo Decal"）
- 作者（默认 = `Configuration.DefaultAuthor`）
- 版本（默认 = `Configuration.DefaultVersion`，初始 "1.0"）
- 描述（多行，默认 = "Generated by SkinTatoo: 包含 N 个 group"）
- TargetGroup 复选框列表（默认勾选所有有图层的 group）

确认后构造 `ModExportOptions` 调 `ModExportService.Export`，期间禁用按钮显示"导出中…"。

### `ModExportService.Export(project, options)` (Services)

主流程（在后台 `Task.Run` 执行）：

1. 创建临时 staging 目录 `Path.Combine(Path.GetTempPath(), "SkinTatoo_Export_" + Guid.NewGuid())`
2. 对每个勾选的 group 调 `previewService.CompositeForExport(group, stagingDir)`，得到 `Dictionary<gamePath, 相对路径>`
3. 合并所有 group 的 redirect 字典 → 构造 `default_mod.json`
4. 构造 `meta.json`（FileVersion=3，字段从 options 取）
5. 调 `pmpPackageWriter.Pack(stagingDir, metaJson, defaultModJson, outputPmpPath)`
6. 若 `Target == InstallToPenumbra`：调 `penumbraBridge.InstallMod(pmpPath)`，然后删除临时 .pmp
7. **finally**：清理 staging 目录
8. 通过 Dalamud Notification 回主线程显示成功 / 失败

### `PreviewService.CompositeForExport(group, stagingDir)` (新增 internal 入口)

```csharp
internal Dictionary<string, string> CompositeForExport(TargetGroup group, string stagingDir)
```

- 复用现有 `LoadBaseTexture` / `CpuUvComposite` / `CompositeEmissiveNorm` / `WriteBgraTexFile`
- **关键差异**：只对 `IsVisible == true` 的图层合成
- emissive：调 `MtrlFileWriter.WriteEmissiveMtrl`，把 `GetCombinedEmissiveColor(visibleLayers)` 烘焙进 .mtrl
- 输出文件落到 `stagingDir/<gamePath>` 的镜像路径
- 返回 `Dictionary<gamePath, 相对磁盘路径>`
- **不**触碰 GPU swap 状态、`baseTextureCache`、`emissiveOffsets`、`previewMtrlDiskPaths`、`EmissiveCBufferHook`

将以下 3 个 `private` 方法改为 `internal`（或保持 private 但通过新 internal 入口委托）：
- `LoadBaseTexture`
- `CpuUvComposite`
- `CompositeEmissiveNorm`

### `PmpPackageWriter` (Services)

```csharp
public static class PmpPackageWriter
{
    public static void Pack(string stagingDir, string metaJson, string defaultModJson, string outputPmpPath);
}
```

- 用 `System.IO.Compression.ZipFile` / `ZipArchive`
- 写入根级 `meta.json` + `default_mod.json`
- 递归写入 staging 目录下所有文件
- zip 内路径分隔符统一为 `/`
- 若 `outputPmpPath` 已存在则覆盖

### `PenumbraBridge.InstallMod(pmpPath)` (Interop)

包装 `Penumbra.Api.IpcSubscribers.InstallMod`，捕获异常返回 `PenumbraApiEc`。

### `Configuration` 新增字段

```csharp
public string DefaultAuthor = "";
public string DefaultVersion = "1.0";
public string LastExportDir = "";
```

## 数据流

```
ModExportWindow (用户填写 + 点确认)
        ↓ ModExportOptions
ModExportService.Export (Task.Run)
        ↓ 对每个勾选 group
PreviewService.CompositeForExport(group, stagingDir)
   ├─ LoadBaseTexture(group)
   ├─ CpuUvComposite(visibleLayers, ...)
   ├─ WriteBgraTexFile → stagingDir/<gamePath_diffuse>.tex
   ├─ if HasEmissiveVisibleLayers:
   │     CompositeEmissiveNorm → stagingDir/<gamePath_normal>.tex
   │     MtrlFileWriter.WriteEmissiveMtrl(bakedColor) → stagingDir/<gamePath_mtrl>.mtrl
   └─ return Dictionary<gamePath, 相对路径>
        ↓ 合并所有 group 的字典
ModExportService 构造 meta.json + default_mod.json
        ↓
PmpPackageWriter.Pack → outputPmpPath (.pmp zip)
        ↓ 若 InstallToPenumbra
PenumbraBridge.InstallMod(installPmpPath) → 立即返回（异步排队）
        ↓ finally
清理 staging 目录（installPmpPath 保留，下次安装覆盖、Dispose 时清理）
        ↓
Dalamud Notification (成功 / 失败 / 部分成功)
```

**重要：Penumbra `InstallMod` IPC 是异步的。** IPC 返回 `Success` 时只是把 import 任务排了队，Penumbra 会在后台 worker 线程才打开 .pmp 文件。所以 install 用的 .pmp **必须**在 IPC 返回后继续存在。我们把它写到插件配置目录 `export_temp/install_pending.pmp` 这个固定路径，下次 install 时被 `PmpPackageWriter.Pack` 自动覆盖，`ModExportService.Dispose()`（插件卸载/游戏退出）时清理。

## .pmp 文件结构

zip 根目录：

```
meta.json
default_mod.json
chara/
  human/
    c0101/
      obj/
        body/
          b0001/
            material/
              mt_c0101b0001_a.mtrl
            texture/
              v01_c0101b0001_d.tex
              v01_c0101b0001_n.tex
  ... (其他 group 的文件按 gamePath 镜像)
```

### `meta.json`（FileVersion 3，参考 `Penumbra/Mods/ModMeta.cs:11-65`）

```json
{
  "FileVersion": 3,
  "Name": "<options.ModName>",
  "Author": "<options.Author>",
  "Description": "<options.Description>",
  "Image": "",
  "Version": "<options.Version>",
  "Website": ""
}
```

### `default_mod.json`（DefaultSubMod 序列化形式）

```json
{
  "Name": "Default",
  "Description": "",
  "Files": {
    "chara/.../v01_c0101b0001_d.tex": "chara/.../v01_c0101b0001_d.tex",
    "chara/.../v01_c0101b0001_n.tex": "chara/.../v01_c0101b0001_n.tex",
    "chara/.../mt_c0101b0001_a.mtrl": "chara/.../mt_c0101b0001_a.mtrl"
  },
  "FileSwaps": {},
  "Manipulations": []
}
```

实施前需到 `Penumbra/Mods/SubMods/DefaultSubMod.cs` 验证精确字段集，确保与当前 Penumbra-CN 的反序列化逻辑一致。

## 错误处理 & 边界情况

### 前置校验（按下导出按钮立即检查，失败弹通知不进入 Export）
- 没有勾选任何 group → "请至少选择一个图层组"
- mod 名为空 → "请输入 Mod 名称"
- 勾选的 group 中没有任何 visible 图层 → "选中的图层组没有可见图层"
- "安装到 Penumbra"路径下 `penumbraBridge.IsAvailable == false` → "Penumbra 未运行"

### 合成阶段（每个 group 独立 try/catch，失败不阻断其他 group）
- `LoadBaseTexture` 返回 null → 记日志，跳过该 group
- `CpuUvComposite` 抛异常 → 同上
- 全部 group 失败 → 整体失败，**不**生成空 .pmp，删除 staging 目录，弹错误通知
- 至少一个 group 成功 → 正常打包，通知里附带 "X 成功 / Y 跳过"

### 打包阶段
- 目标 .pmp 已存在 → 正常覆盖
- 目标目录无写权限 / 磁盘满 → 捕获 `IOException` 弹错误通知
- ZipFile 写入失败 → 同上

### Penumbra 安装阶段
- `InstallMod` 返回非 `Success` → 把 `PenumbraApiEc` 写进通知（"安装失败：<EcCode>"）
- IPC 抛异常 → 捕获后通知失败
- **不要在 finally 中删除 install pmp**：IPC 异步排队，Penumbra 真正读文件发生在 IPC 返回之后

### 清理
- **staging 目录**：finally 块中始终删除，即使失败
- **install_pending.pmp**：写到 `<pluginConfigDir>/export_temp/install_pending.pmp`，由 `ModExportService.Dispose()` 在插件卸载时清理；下次安装会被 `PmpPackageWriter.Pack` 自动覆盖；构造函数也清一次以收尾上次会话崩溃残留

### 线程模型
- 整个 `Export` 流程在后台 `Task.Run` 执行（合成耗时秒级）
- 期间禁用导出按钮，防止重复点击
- 完成后通过现有 Notification 通道回到主线程提示
- **不**触碰 PreviewService 的 GPU swap / hook 状态，与正在运行的实时预览互不干扰

### 日志
- 每步通过 `DebugServer.AppendLog("[ModExport] ...")` 记录
- 可经 HTTP `/api/log` 查询调试
