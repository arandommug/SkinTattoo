# 材质替换路线研究 — mask / 金属化 / 石化

> 2026-04-07 调研，源材料：Penumbra-CN、Glamourer-CN、本项目现有 hook/swap 实现。

## 起因

用户提问："能不能改 mask？mask 是材质吗？能不能把材质改成金属/石化？"
示例：`chara/human/c1401/obj/body/b0001/material/v0001/mt_c1401b0001_b.mtrl` + `chara/nyaughty/eve/gen3_aura_mask.tex`。

## 概念厘清

| 术语 | 文件 | 说明 |
|---|---|---|
| **材质** | `.mtrl` | shader 配置 + sampler 列表 + 常量 + 可选 ColorTable |
| **mask** | `.tex` | 普通纹理，被 `.mtrl` 通过 sampler 索引引用 |

`gen3_aura_mask.tex` 是 Nyaughty Eve/gen3 体型 mod 自带的辅助贴图。**Vanilla `skin.shpk` 不采样 mask**——只读 diffuse + normal。所以这张 tex 一定是 mod 把 shader 切到带 mask 通道的扩展 skin shader 之后才被采样的。

## 关键事实校验

### 事实 1：vanilla skin.shpk 仍然没有 ColorTable
`Glamourer/Interop/Material/PrepareColorSet.cs:81-88`
```csharp
public static bool TryGetColorTable(MaterialResourceHandle* material, ...)
{
    if (material->DataSet is null
        || material->DataSetSize < sizeof(ColorTable.Table)
        || !material->HasColorTable)
    {
        table = default;
        return false;
    }
```
Glamourer 自己用 `HasColorTable` 短路，对 `mt_c1401b0001_b.mtrl` 返回 false。**ColorTable 路线不能直接用于身体皮肤**。我们之前在 `CLAUDE.md` 写的"skin.shpk 无 ColorTable"结论依然成立。

### 事实 2：Dawntrail 后 ColorTableRow 增加了原生 Roughness / Metalness 字段
`Penumbra.GameData/Files/MaterialStructs/ColorTableRow.cs:7-19`
```
#       |    X (+0)    |    Y (+1)    |    Z (+2)    |    W (+3)    |
0 (+ 0) |    Diffuse.R |    Diffuse.G |   Diffuse.B  |     Unk      |
1 (+ 4) |   Specular.R |   Specular.G |  Specular.B  |     Unk      |
2 (+ 8) |   Emissive.R |   Emissive.G |  Emissive.B  |     Unk      |
3 (+12) |   Sheen Rate |   Sheen Tint |  Sheen Apt.  |     Unk      |
4 (+16) |  Roughness?  |              |  Metalness?  |  Anisotropy  |
5 (+20) |          Unk |  Sphere Mask |          Unk |          Unk |
6 (+24) |   Shader Idx |   Tile Index |  Tile Alpha  |  Sphere Idx  |
7 (+28) |   Tile XF UU |   Tile XF UV |  Tile XF VU  |  Tile XF VV  |
```
- 每行 = 8 vec4 = **32 个 Half = 64 字节**
- 整张表 32 行 × 64 字节 = **2048 字节**（旧 legacy 是 16 行 × 16 字节 = 256 字节）
- Half offset 速查：
  - SpecularColor `[4][5][6]`
  - EmissiveColor `[8][9][10]`
  - SheenRate `[12]` / SheenTint `[13]` / SheenAperture `[14]`
  - **Roughness `[16]`**
  - **Metalness `[18]`**

### 事实 3：Dawntrail vs Legacy 仅靠 ShaderPackage 名字判别
`Glamourer/Interop/Material/PrepareColorSet.cs:144-149`
```csharp
public static ColorRow.Mode GetMode(MaterialResourceHandle* handle)
    => handle == null ? ColorRow.Mode.Dawntrail
        : handle->ShpkName.AsSpan().SequenceEqual("characterlegacy.shpk"u8)
            ? ColorRow.Mode.Legacy
            : ColorRow.Mode.Dawntrail;
```
只有 `characterlegacy.shpk` 走 legacy 布局，其它（含 `character.shpk`/`skin.shpk`/`hair.shpk`/`iris.shpk`）都按 Dawntrail 布局。

### 事实 4：Glamourer 的 ColorTable 实时替换是完整范本
| 操作 | 文件 | 行 |
|---|---|---|
| GPU → CPU 读 ColorTable | `Glamourer/Interop/Material/DirectXService.cs` | 49-154 (D3D11 staging + memcpy) |
| CPU → GPU 写 ColorTable | `Glamourer/Interop/Material/DirectXService.cs` | 21-47 (R16G16B16A16Float) |
| 实时改一行并下发 | `Glamourer/State/StateApplier.cs` | 335-356 `ChangeMaterialValue` |
| 单行写入 | `Glamourer/Interop/Material/MaterialValueManager.cs` | 79-155 `ColorRow.Apply` |

我们的 `TextureSwapService.UpdateEmissiveViaColorTable` 走的是同一条路，只是只写 emissive 字段。

### 事实 5：Penumbra 提供完整的 .mtrl 增删改 API
| 能力 | 文件 |
|---|---|
| 解析 | `Penumbra.GameData/Files/MtrlFile.cs:154-217` |
| 序列化 | `Penumbra.GameData/Files/MtrlFile.Write.cs:7-90` |
| `FindOrAddShaderKey` | `Penumbra.GameData/Files/MtrlFile.AddRemove.cs:103-126` |
| `FindOrAddConstant` / `GetConstantValue<T>` | `Penumbra.GameData/Files/MtrlFile.AddRemove.cs:12-51` |
| `FindOrAddSampler` | `Penumbra.GameData/Files/MtrlFile.AddRemove.cs:53-90` |
| ColorTable UI 编辑器 | `Penumbra/UI/FileEditing/Materials/MaterialEditor.ColorTable.cs` |
| ShaderPackage 切换 UI | `Penumbra/UI/FileEditing/Materials/MaterialEditor.ShaderPackage.cs` |

我们当前的 `MtrlFileWriter.RebuildMtrl` 是 emissive-only 的简化版，未来如果要支持完整材质改写应直接用 Penumbra 的 `MtrlFile`。

### 事实 6：Penumbra/Glamourer 都没有"运行时 mask 通道编辑"功能
`Penumbra/Import/Models/Export/MaterialExporter.cs:83-93` 仅在 glTF 导出时按 `TextureUsage.SamplerMask` 拆通道（R=AO，A=Specular factor）。两个项目都没有 mask 编辑 UI——这块如果要做需要自己合成。

## 路线对比

| 目标 | 路线 | 复用什么 | 工作量 | 备注 |
|---|---|---|---|---|
| 装备/眼/发金属化或石化 | **A. 扩展 ColorTable swap** 写 Metalness/Roughness/SpecularColor | 现有 `UpdateEmissiveViaColorTable` + Penumbra `ColorTableRow` 字段表 | **小** | **首选起点** |
| vanilla 身体（skin.shpk）金属化 | B. EmissiveCBufferHook 同款手段 hook OnRenderMaterial 改 g_Specular*/g_Material* CBuffer 常量 | 现有 hook，需要逆向 skin.shpk 常量 CRC 含义 | 中 | skin shader 风格写死，效果上限低 |
| 身体彻底换风格 | C. 用 Penumbra `MtrlFile` 重写 `.mtrl`，把 `ShaderPackageName` 从 skin.shpk 换 character.shpk + 重建 sampler/ColorTable + 走 Penumbra 临时 mod | Penumbra `MtrlFile` + `MtrlFile.AddRemove` | 大 | 容易崩，但有现成轮子 |
| 改 mask `.tex` 内容 | D. 现有 `TextureSwapService.SwapTexture` 直接换 GPU 纹理 | 现有 swap 通道 | 小 | **仅当当前材质真的引用了 mask sampler 才生效** |

## 决策

**先做路线 A（最小可验证实验）**：在 `TextureSwapService` 加 `UpdateMaterialPbr(metalness, roughness, specularColor)`，对 character.shpk 类材质用 ColorTable swap 写 Dawntrail 的 Metalness/Roughness/SpecularColor。先在装备/头发上验证可行，再决定是否对身体投入更大工程。

## Latent bug 提醒

`TextureSwapService.UpdateEmissiveViaColorTable` 的 `int rowStride = ctWidth * 4`：
- emissive 在每行 +8/+9/+10，无论 8 vec4/行 (Dawntrail) 还是 4 vec4/行 (legacy) 都正确（因为 emissive 在 vec4 #2 内）
- 但**写 Metalness (+18) 和 Roughness (+16) 时只对 Dawntrail 8-vec4 布局有效**
- 实现 `UpdateMaterialPbr` 时必须先用 `ctWidth >= 8` 区分布局，legacy 模式下要走 `Scalar7` (SpecularMask) + `Scalar3` (Shininess) 而不是 Metalness/Roughness
